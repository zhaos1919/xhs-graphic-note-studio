请用 Python Pillow 将以下文字直接渲染到我上传的底图上，生成排版图片。

一、基础要求
- 底图：使用我上传的图片作为底图，不要重新生成背景
- 图片分辨率：保持原底图尺寸不变
- 字体：Noto Serif CJK SC（思源宋体）
- 文字颜色：#1a1a1a

二、字号与字重
- 章节标题：Bold，默认 50px
- 条目小标题（加粗词条）：Bold，默认 36px
- 正文：Regular，默认 34px

三、排版参数
- 左右边距：95px
- 顶部起始：85px
- 底部安全边距：80px
- 正文默认行高：56px
- 标题默认行高：76px
- 小标题默认行高：54px
- 条目之间默认间距：6px
- 组与组之间默认额外间距：16px

四、内容结构说明
请根据实际内容自动识别并排版：
- heading：章节大标题，加粗大字
- item_title：条目名称，加粗，紧接正文
- item_body：条目正文，常规字重，自动换行
- label：对比类小标签（如“普通写法：”）
- body：对比类正文
- divider：分隔线（用于对比组之间）
- spacer：空白间隔

五、强制排版规则
1. 渲染前必须先计算总高度，确保所有文字都在图片底部安全边距（80px）以内，绝不能超出图片。
2. 如内容过多放不下，按以下顺序自动调整：
   - 先缩减各类间距与行高，但正文/小标题行高最低不低于48px
   - 再缩小字号，但正文、条目标题、章节标题最小不低于30px
3. 每次生成后必须打印：
   - final y
   - bottom_margin
   用于确认底部留白充足后再输出图片。
4. 每页都要独立进行高度计算，不得让文字溢出到底图外。
5. 自动换行时必须按当前可用宽度精确计算，不要按字符数粗略截断。
6. 保持整体风格简洁、疏朗、规整，不要额外添加图形、边框、装饰元素。
7. 在不超出安全边距的前提下，优先让文字与底图画面显得协调、饱满；如果底图留白较多，可适度放大正文与条目标题，使画面更和谐，但仍要保证层级清晰、版面不过挤。

六、条目正文的特殊规则
1. 条目标题 + 正文描述部分可以同起一行排版。
2. 如果正文中出现“适配”二字：
   - 必须将“适配……”这一段整体另起一行
   - “适配……”这一段需要单独按整行宽度自动换行
   - “适配……”这一段结束后，必须再额外空一整行，再开始下一个条目
3. 条目正文必须拆分为两部分处理：
   - 前半段描述文字
   - 从“适配”开始的适配说明
4. 前半段描述按“条目标题 + 正文”排版；
   从“适配”开始必须强制断到下一行，不得继续接在首行后面。

七、对比页特殊规则
1. “普通写法：”“优化写法：”必须作为加粗 label。
2. label 后正文紧接排版，并按可用宽度自动换行。
3. 每一组“普通写法 + 优化写法”视为一个完整分组。
4. 每组内部：
   - 普通写法 与 优化写法 之间保留较小间距。
5. 每组结束后：
   - 必须额外空一整行，再开始下一组。
6. 对比页整体要比普通正文页更注意节奏感和呼吸感，避免四组内容粘连在一起。

八、输出要求
- 直接输出生成后的图片文件
- 如果正文被分成多页，则分别输出每一页图片
- 同时给出全部图片的下载路径
- 如果有多页，额外打包成 zip

九、代码要求
- 必须使用 Python + Pillow 实现
- 必须实际生成图片文件，不要只给示例代码
- 代码里必须包含：
  - 字体加载
  - 自动换行
  - 高度预计算
  - 超高时自动压缩间距/字号
  - “适配”段单独换行逻辑
  - “适配”段结束后空一整行逻辑
  - 对比页每组结束后空一整行逻辑
  - final y 与 bottom_margin 打印
  - 多页输出保存
